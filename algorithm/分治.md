# 分治（Divide and Conquer）

分治的核心思想是把原问题拆成若干个规模更小、结构相同的子问题，先递归解决子问题，再把子问题结果合并成原问题答案。

典型流程可以写成三步：

1. 分解（Divide）：把规模为 `n` 的问题拆成 `a` 个规模约为 `n / b` 的子问题；
2. 解决（Conquer）：递归求解这些子问题；
3. 合并（Combine）：把子问题答案组合成原问题答案。

## 适用条件

- 子问题和原问题结构一致；
- 子问题相互独立，重叠很少；
- 合并步骤有清晰、可实现的规则。

如果子问题高度重叠，通常更适合动态规划；如果每一步只保留一个局部最优选择，通常更接近贪心。

## 复杂度分析：主定理

分治递归常写为：

`T(n) = aT(n / b) + f(n)`

- `a`：子问题个数；
- `b`：子问题规模缩小倍数；
- `f(n)`：分解与合并代价。

关键比较对象是 `n^(log_b a)` 与 `f(n)`，由此判断总复杂度主导项。

## 常见算法

### 归并排序

- 分解：数组一分为二；
- 合并：线性合并两个有序数组；
- 复杂度：`O(n log n)`，稳定排序，额外空间 `O(n)`。

### 快速排序

- 分解：按基准值 partition；
- 合并：隐式完成（基准归位）；
- 平均复杂度 `O(n log n)`，最坏 `O(n^2)`；
- 原地排序，空间主要来自递归栈。

### 二分查找

- 每次把搜索区间减半；
- 复杂度 `O(log n)`；
- 前提是数据有序且区间边界定义严格。

### 快速幂

- 利用 `a^n` 在 `n` 为偶数/奇数时的递推关系；
- 复杂度从 `O(n)` 降到 `O(log n)`；
- 常用于大数取模、矩阵快速幂。

## 分治写题模板

```text
solve(problem):
    if problem 足够小:
        return 直接解
    left, right, ... = split(problem)
    ans1 = solve(left)
    ans2 = solve(right)
    return merge(ans1, ans2, ...)
```

## 经典题型

- 排序与选择：归并排序、快速排序、数组第 k 小（QuickSelect）；
- 区间处理：线段树的构建与查询可看作分治思想；
- 计算几何：最近点对；
- 大整数运算：Karatsuba 乘法；
- 树问题：递归求子树信息再合并到父节点。

## 常见错误

- 递归边界不完整，导致无限递归；
- 分解后子区间重叠或遗漏；
- 合并阶段破坏有序性或重复计算；
- 对快速排序固定选端点当 pivot，导致近乎有序数据退化。

## 与动态规划的区别

- 分治：子问题通常独立，不强调记忆化；
- 动态规划：子问题大量重叠，需要缓存结果避免重复计算。

面试中一句话区分：分治是“拆开分别算再合起来”，动态规划是“重叠子问题只算一次”。
