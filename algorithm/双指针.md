# 双指针（Two Pointers）

双指针是通过维护两个位置索引（或两个迭代器）来降低时间复杂度的技巧，常把朴素 `O(n^2)` 降到 `O(n)` 或 `O(n log n)`。

## 常见形态

### 1) 相向双指针

两个指针分别从区间两端向中间移动，常用于有序数组。

- 典型问题：两数之和（有序数组）、三数之和、盛最多水的容器、回文判断。

### 2) 同向快慢指针

两个指针从左到右移动，但速度或含义不同。

- 典型问题：原地去重、移除元素、链表找中点、判断链表是否有环。

### 3) 滑动窗口（本质是双指针）

维护窗口 `[l, r]`，右指针扩展、左指针收缩。

- 典型问题：最长无重复子串、最小覆盖子串、和为 K 的子数组变体。

## 题型与要点

### 有序数组两数和

- 初始化 `l = 0, r = n - 1`；
- 若 `a[l] + a[r]` 太大，`r--`；太小，`l++`；
- 正确性依赖单调性，复杂度 `O(n)`。

### 原地去重（有序数组）

- 慢指针指向“已处理区间末尾”；
- 快指针扫描新元素；
- 当出现新值时写入慢指针下一位。

### 链表快慢指针

- 找中点：快指针每次 2 步，慢指针每次 1 步；
- 判环：Floyd 判圈，若有环快慢会相遇；
- 定位环入口：相遇后一个指针回头，同速前进再次相遇即入口。

### 滑动窗口通用框架

```text
l = 0
for r in [0..n-1]:
    扩展窗口，更新状态
    while 窗口不满足约束:
        收缩 l，回收状态
    用当前窗口更新答案
```

## 复杂度直觉

- 每个指针最多走 `n` 步，因此总移动次数是线性级；
- 滑动窗口中的内层 `while` 并不一定导致 `O(n^2)`，因为 `l` 只会单调增加。

## 何时不能直接用双指针

- 无序且没有可利用单调性；
- 状态不具备“可增可减”维护能力；
- 需要回溯历史多分支决策。

这时通常考虑排序预处理、哈希、前缀和或动态规划。
