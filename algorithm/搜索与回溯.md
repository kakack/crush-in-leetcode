# 搜索与回溯（Search and Backtracking）

搜索的本质是“在状态空间里找满足条件的状态”。常见策略是 DFS、BFS、回溯、剪枝。

## 1. DFS 与 BFS

### DFS

- 沿一条路径走到底再回退；
- 适合枚举所有解、路径构造、连通块遍历；
- 实现方式：递归或显式栈。

### BFS

- 按层扩展；
- 适合最短步数（无权图）问题；
- 实现方式：队列。

## 2. 回溯框架

回溯可以理解为“带撤销操作的 DFS”，用于生成排列、组合、子集、棋盘放置、分割等问题。

```text
backtrack(状态):
    if 到达终止条件:
        记录答案
        return
    for 每个可选决策:
        做选择
        backtrack(新状态)
        撤销选择
```

## 3. 三要素：路径、选择列表、结束条件

- 路径：当前已经做出的决策；
- 选择列表：当前状态下还可以做哪些选择；
- 结束条件：何时收集答案并返回。

大多数回溯错误都来自这三项定义不清。

## 4. 常见剪枝

- 顺序剪枝：先排序，利用单调性提前停止；
- 去重剪枝：同层去重，避免等价状态重复搜索；
- 可行性剪枝：当前代价已不可能优于最优解；
- 约束剪枝：一旦违反约束立即返回。

剪枝不改变正确性，但决定能否在时限内通过。

## 5. 经典题型

### 子集 / 组合

- 子集：每个元素选或不选；
- 组合：从候选集合中选择固定数量；
- 常用“从 start 开始枚举”，避免重复顺序。

### 排列

- 需要 `used[]` 记录哪些元素已选；
- 对含重复元素数组，先排序并做同层去重。

### N 皇后

- 逐行放置皇后；
- 约束为列、主对角线、副对角线不冲突；
- 使用集合或位运算优化冲突检测。

### 网格搜索

- 方向数组 `dirs`；
- 注意边界检查和 visited 标记；
- 题型：岛屿数量、单词搜索、最短路径。

## 6. 复杂度认知

- 回溯本质通常是指数级或阶乘级；
- 真正可做的是：降常数、强剪枝、快速判冲突；
- 写题时优先估算分支因子与深度，再决定是否可行。

## 7. 易错点

- 忘记撤销状态；
- visited 标记时机错误（入栈标记 vs 出栈标记）；
- 去重维度错误（同层去重与同枝去重混淆）；
- BFS 忘记分层，导致步数统计错误。
