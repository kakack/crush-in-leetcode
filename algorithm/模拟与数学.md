# 模拟与数学

这类题的共同点是：不一定依赖复杂数据结构，但非常依赖细节正确性与边界处理。

## 一、模拟题

模拟就是把题目过程按规则“原样执行”。

### 常见场景

- 矩阵/网格移动：方向数组、边界判定、状态更新；
- 字符串解析：按字符流处理、有限状态机；
- 日历与时间：进位、闰年、跨月；
- 自定义数据结构行为题：按接口逐步实现。

### 写模拟题的顺序

1. 定义状态变量；
2. 写一步转移；
3. 写循环与终止条件；
4. 用样例手推校验。

## 二、数学题

数学题常考“公式 + 数论 + 不变量”。

### 高频知识点

- 最大公约数与最小公倍数：`gcd(a, b)`、`lcm(a, b) = a / gcd(a,b) * b`；
- 质数与筛法：埃氏筛；
- 快速幂：`a^n mod m`；
- 进制转换与位权；
- 组合计数：排列组合、容斥、抽屉原理；
- 同余与模运算。

### 常见技巧

- 防溢出：先除后乘，或用 `long long`；
- 结果取模：每一步都取模而不是最后一次；
- 浮点误差：比较时用误差阈值。

## 三、LeetCode 题型映射

- 模拟：54 Spiral Matrix、59 Spiral Matrix II、6 Zigzag Conversion；
- 数学：7 Reverse Integer、9 Palindrome Number、50 Pow(x, n)；
- 数论：202 Happy Number、204 Count Primes、279 Perfect Squares；
- 计数/组合：62 Unique Paths、1359 Count All Valid Pickup and Delivery Options。

## 四、易错点

- 状态更新顺序错误；
- 边界条件漏掉空输入、单元素、负数；
- 模运算与除法混用；
- 把“可以证明的规律”误当成“拍脑袋规律”。
