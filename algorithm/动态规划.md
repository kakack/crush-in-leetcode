# 动态规划（DP）

动态规划用于解决“最优化/计数”问题：把原问题拆成子问题，并保存子问题答案避免重复计算。

## 何时考虑 DP

- 存在最优子结构：原问题最优解可由子问题最优解组合得到；
- 存在重叠子问题：相同子问题会被反复求解；
- 状态可被有限维度表达：如位置、容量、剩余次数、是否贴边界等。

## DP 四步法

1. 定义状态：`dp[i]`、`dp[i][j]` 的含义；
2. 写转移：当前状态如何由更小状态推导；
3. 设初值：边界状态与非法状态；
4. 定顺序：确保转移依赖的状态先被计算。

## 与分治、贪心的区别

- 分治：子问题通常相互独立，强调拆分与合并；
- 贪心：每一步选局部最优且不回头，前提是可证明贪心选择性质；
- 动态规划：会系统比较多种子状态，保证全局最优。

# 常见题型

- 状态定义：定义子问题，常用dp[n]数组的形式表示，dp[i]就是表示规模为i的问题的解；
- 状态转移：子问题之间的关系，常写作dp[n]=f(dp[i])， 其中i<n。

## 1. 线性动态规划

线性动态规划的主要特点是状态的推导是按照问题规模 i 从小到大依次推过去的，较大规模的问题的解依赖较小规模的问题的解。

- 状态定义：dp[n] := [0..n] 上问题的解
- 状态转移：dp[n] = f(dp[n-1], ..., dp[0])

### 1.1 单串

单串线性 DP 的典型形式：`dp[i]` 表示以位置 `i` 结尾（或考虑 `[0..i]`）的某种最优解或方案数。

**常见模式：**

- **最长递增子序列（LIS）**：`dp[i] = max{ dp[j] + 1 : j < i 且 a[j] < a[i] }`，最终答案为 `max(dp[0..n-1])`。
- **最大子数组和**：`dp[i]` 表示以 `i` 结尾的最大和，`dp[i] = max(nums[i], dp[i-1] + nums[i])`，可压成单个变量。
- **爬楼梯 / 斐波那契**：`dp[i] = dp[i-1] + dp[i-2]`，只依赖前两项时可滚动数组。

**LeetCode 代表题**：53-Maximum Subarray、300-Longest Increasing Subsequence、70-Climbing Stairs。

### 1.2 双串

双串线性 DP：状态与两个序列的「长度」或「下标」相关，常用 `dp[i][j]` 表示第一个串前 `i` 个、第二个串前 `j` 个的某种最优解。

**常见模式：**

- **最长公共子序列（LCS）**：若 `s1[i-1] == s2[j-1]` 则 `dp[i][j] = dp[i-1][j-1] + 1`，否则 `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。
- **编辑距离**：插入、删除、替换对应 `dp[i][j]` 从 `dp[i][j-1]`、`dp[i-1][j]`、`dp[i-1][j-1]` 转移，取 min 再加 1（若当前字符相同则 `dp[i-1][j-1]` 不加 1）。

**LeetCode 代表题**：1143-LCS、72-Edit Distance、10-Regular Expression Matching。

### 1.3 矩阵

矩阵 DP 中状态通常为「走到 (i, j) 的某种最优解」，转移来自上方或左方（有时左上、右方），注意边界。

**常见模式：**

- **最小路径和**：`dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`，可原地或滚动一维。
- **最大正方形 / 矩形**：以 `(i,j)` 为右下角的正方形边长可由 `dp[i-1][j]`、`dp[i][j-1]`、`dp[i-1][j-1]` 推导。

**LeetCode 代表题**：64-Minimum Path Sum、221-Maximal Square、120-Triangle。

## 2. 前缀和

前缀和将「区间和」转化为「两个前缀和相减」，从而在 O(1) 时间内回答多次区间和查询。

**一维**：`pre[i] = a[0] + a[1] + ... + a[i-1]`，区间 `[l, r]` 和为 `pre[r+1] - pre[l]`。

**二维**：`pre[i][j]` 为从 (0,0) 到 (i-1,j-1) 的矩形和，则子矩阵 (r1,c1)-(r2,c2) 和为  
`pre[r2+1][c2+1] - pre[r1][c2+1] - pre[r2+1][c1] + pre[r1][c1]`。

### 2.1 数据结构维护前缀和

若元素会动态变化，可用树状数组（Binary Indexed Tree）或线段树维护前缀和，支持单点修改与区间查询，均为 O(log n)。

### 2.2 运算推广

前缀和思想可推广到其它满足「可减性」的运算（即知道区间两端的前缀值就能算区间结果）：

- **异或前缀和**：`pre[i] = a[0]^a[1]^...^a[i-1]`，区间 `[l,r]` 的异或为 `pre[r+1]^pre[l]`。用于「子数组异或」类题目。
- **前缀积**：若需取模、且模意义下有逆元，可做前缀积并做区间「除」得到区间积。

### 2.3 差分

差分是前缀和的逆：对数组 `d` 做前缀和得到原数组 `a`。对 `a` 的区间 `[l,r]` 统一加 `c`，等价于 `d[l] += c`，`d[r+1] -= c`，最后再对 `d` 做一次前缀和还原 `a`。常用于「多次区间修改、最后问整个数组」的问题。

## 3. 区间

区间 DP 的状态通常定义为「区间 [i, j] 上的最优解」，按区间长度从小到大递推。

- **状态**：`dp[i][j]` 表示区间 `[i, j]`（或 `nums[i..j]`）的某种最优解。
- **转移**：枚举区间内分割点 k，`dp[i][j] = f(dp[i][k], dp[k+1][j])`，或根据首尾元素是否参与决策转移。

**例如**：最长回文子串、戳气球、合并石子、矩阵链乘法等。注意循环顺序：先枚举区间长度 len，再枚举起点 i，则 `j = i + len - 1`。

## 4. 背包

背包问题是「在容量限制下选择物品使价值最大（或方案数）」的模型。

- **0-1 背包**：每件物品最多选一次。`dp[j]` 表示容量 j 的最大价值，逆序枚举 j（一维）从 `dp[j - w[i]] + v[i]` 转移。
- **完全背包**：每件物品可无限选。一维时正序枚举 j。
- **多重背包**：每件物品有数量上限，可二进制拆分或单调队列优化。

**状态设计**：`dp[i][j]` 表示前 i 个物品、容量 j 的最优解；压维后 `dp[j]` 表示容量 j 的最优解。

## 5. 状态压缩

当状态中「每一维只有少量取值」（如布尔、小整数）时，可用整数二进制位表示状态，从而用 `dp[1<<n]` 等表示「某个子集」对应的最优解。

**例如**：旅行商（TSP）、任务分配、棋盘覆盖等。`dp[mask][i]` 表示已访问集合为 mask、当前在 i 的最优解；转移时枚举下一个未访问点 j。

## 6. 计数

计数 DP 与最优化 DP 类似，但状态存的是方案数，转移时用加法（或乘法）组合子问题方案数，注意取模和边界（如空集为 1）。

**例如**：不同二叉搜索树个数、括号序列计数、数字 1 的个数等。

## 7. 矩阵快速幂

若状态转移可写成线性递推（如 `dp[n] = c1*dp[n-1] + c2*dp[n-2] + ...`），可把递推写成矩阵乘法，用矩阵快速幂在 O(k³ log n) 内求第 n 项，其中 k 为状态维度。

**例如**：斐波那契、爬楼梯变种、状态机类计数（如「不能连续出现 1」的 01 串个数）。

## 8. 数位

数位 DP 求的是「在区间 [L, R] 内满足某性质的数的个数」，通常转化为 `count(R) - count(L-1)`，再对「不超过 N 的满足条件的数」按数位从高到低 DP。

- **状态**：`dp[pos][tight][state]` 表示当前处理到第 pos 位、是否紧贴上界、以及题目相关的状态（如是否已出现某数字、模某数余数等）。
- **转移**：枚举当前位数字，注意上界和前导零。

**例如**：不含 4 的数字个数、数字 1 的个数、各数位之和等。
