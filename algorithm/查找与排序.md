# 查找与排序

这一章按 LeetCode 实战来组织，不追求算法“全家桶”，而是强调：什么时候该用哪种排序/查找模型。

## 一、先记结论

- 业务刷题默认：`sort` + 双指针 / 二分；
- 需要稳定排序：归并或 `stable_sort`；
- 需要原地且平均快：快速排序；
- 需要 TopK：堆（`priority_queue`）或快速选择；
- 值域小的整数：计数排序；
- 数据近乎有序：插入排序通常很快。

## 二、查找模型

### 1) 顺序查找

- 无序数组的基线方法，复杂度 `O(n)`。

### 2) 二分查找

- 前提：序列有序或答案空间具备单调性；
- 模板目标：找“第一个满足条件的位置”；
- 高频：34、35、69、704、875、1011。

### 3) 哈希查找

- 典型 `O(1)` 均摊；
- 高频：1、49、128、347。

# 排序算法速览

| 类别 | 代表 | 平均复杂度 | 稳定性 |
|---|---|---|---|
| 比较排序 | 快排、归并、堆排 | `O(n log n)` | 归并稳定 |
| 简单排序 | 冒泡、选择、插入 | `O(n^2)` | 插入/冒泡稳定 |
| 非比较排序 | 计数、基数、桶排 | 与值域相关 | 常可稳定 |

简单排序在 LeetCode 中更多用于理解“局部有序”和“原地调整”，真正写题常用标准库排序。

## 三、LeetCode 常用排序策略

### 1) 默认首选：标准库排序

- C++：`sort(nums.begin(), nums.end())`，平均 `O(n log n)`；
- Python：`nums.sort()`（Timsort），对部分有序数据表现很好；
- 之后常接：双指针、分组扫描、去重、二分。

### 2) 需要 TopK

- 小顶堆维护 K 个元素：`O(n log k)`；
- 快速选择（QuickSelect）：平均 `O(n)`，最坏 `O(n^2)`。

### 3) 需要稳定性

- 归并排序或稳定库排序（如 `stable_sort`）；
- 典型场景：多关键字排序，先按次关键字排，再按主关键字稳定排序。

## 四、关键算法模板（刷题够用）

### 快速排序（平均快，原地）

```C++
int partition(vector<int>& a, int l, int r) {
    int pivot = a[r];
    int i = l;
    for (int j = l; j < r; ++j) {
        if (a[j] <= pivot) swap(a[i++], a[j]);
    }
    swap(a[i], a[r]);
    return i;
}
```

- 平均 `O(n log n)`，最坏 `O(n^2)`；
- 空间主要是递归栈，平均 `O(log n)`；
- 不稳定。

### 归并排序（稳定）

- 分治 + 合并；
- 时间 `O(n log n)`，空间 `O(n)`；
- 高频于“统计逆序对、区间计数”等题型改造。

### 堆排序 / 堆结构

- 排序时间 `O(n log n)`，空间 `O(1)`（原地堆排）；
- 刷题更常见的是“堆结构”而不是手写堆排：TopK、流式中位数、合并 K 路。

### 非比较排序

- 计数排序：`O(n + k)`，适合值域 `k` 不大；
- 基数排序：按位稳定排序，适合非负整数或定长串；
- 桶排序：依赖分布假设，工程中使用相对少。

## 五、复杂度与选择速查

| 场景 | 优先方案 |
|---|---|
| 普通排序 | 标准库 `sort` / `list.sort` |
| 稳定排序 | 归并 / `stable_sort` |
| TopK | 堆 或 快速选择 |
| 值域很小 | 计数排序 |
| 有序数组查找 | 二分查找 |
