# 贪心算法

贪心算法是对完成一件事情的方法的描述：**每一步都做出当前看起来最好的选择，且不回溯**，期望通过一系列局部最优得到全局最优。

## 适用场景

- 问题可以分解为多步，每一步有若干选择。
- 与回溯、动态规划的区别：
  - **回溯**：需要记录每一步、每一种选择，用于枚举所有具体解。
  - **动态规划**：记录的是每一步、所有选择汇总后的值（最大、最小或计数），子问题可能重叠，需要保存多种局部状态。
  - **贪心**：在适用的问题中，每一步选择“当前最优”且不回头，一般只维护当前决策所需状态，不保存完整历史状态。

## 正确性条件

能用贪心得到全局最优解，通常需满足：

1. **最优子结构**：大问题的解由小问题的解组成。与 DP 不同，贪心问题中「大问题的解」往往只由**某一个**「小问题的解」直接决定（即选了一条路就不回头）。
2. **无后效性**：后续决策不会改变之前已经确定的结果。
3. **贪心选择性质**：**局部最优能推出全局最优**。这是贪心与 DP 的关键区别——必须证明「每一步选当前最优」不会导致丢失全局最优解。证明方法常用：交换论证、归纳法、反证法（假设存在更优解，证明可经有限次交换调整为贪心解且不更差）。

## 常见题型与策略

- **区间问题**：按右端点（或左端点）排序，依次选不重叠的区间（如 435 无重叠区间、452 用最少箭引爆气球）。
- **分配问题**：排序 + 双指针或贪心匹配（如 455 分发饼干、135 分发糖果）。
- **跳跃与覆盖**：维护当前能到达的最远位置，在可达范围内选下一跳（如 55 跳跃游戏、45 跳跃游戏 II）。
- **字典序**：栈上贪心保留「尽量小」的字符（如 402 移掉 K 位数字、316 去除重复字母）。
- **Huffman / 合并**：每次选当前代价最小的两堆合并（如 1167 连接棒材的最低费用）。

## 小结

贪心代码短、效率高，但**必须保证贪心选择性质成立**；若无法证明，应考虑 DP 或回溯。做题时先想「若每一步都选当前最优，会不会漏掉全局最优？」再用交换论证或归纳验证。
